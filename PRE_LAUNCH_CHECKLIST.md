## ✅ 必要項目 (CRITICAL)

這些項目如果缺失，App 會被 Google Play 拒絕或導致嚴重問題。

### 1. 🔐 隱私政策 (Privacy Policy)

- [ ] **撰寫隱私政策**
  - 說明收集哪些數據（即使只是本地存儲）
  - 說明數據如何使用
  - 提供聯絡方式
  - **狀態：❌ 缺失**
  
- [ ] **託管隱私政策**
  - 上傳到公開網址（GitHub Pages、Netlify 等）
  - 在 Google Play Console 中提供連結
  - **狀態：❌ 缺失**

**建議：**
```markdown
# 隱私政策範本
本遊戲不收集任何個人資料。
所有遊戲進度存儲在您的裝置本地。
我們不使用分析工具或第三方追蹤。
聯絡方式：your-email@example.com
```

### 2. 📱 App 商店資產 (Store Assets)

- [ ] **App 圖示 (Icon)**
  - 尺寸：512x512 px（高解析度）
  - 格式：PNG（32-bit）
  - 無透明背景
  - **目前狀態：✅ 有 `icon.png`，需確認尺寸**

- [ ] **Feature Graphic（主視覺圖）**
  - 尺寸：1024x500 px
  - 格式：PNG 或 JPG
  - 展示遊戲特色
  - **狀態：❌ 缺失**

- [ ] **截圖（Screenshots）**
  - 至少 2 張，建議 4-8 張
  - 手機：1080x1920 px（9:16）或實際螢幕比例
  - 展示遊戲核心玩法
  - **狀態：❌ 缺失**

- [ ] **短描述（Short Description）**
  - 最多 80 字元
  - 吸引玩家的一句話
  - **狀態：❌ 缺失**

- [ ] **完整描述（Full Description）**
  - 詳細介紹遊戲玩法、特色
  - 最多 4000 字元
  - **狀態：❌ 缺失**

**建議短描述：**
```
放置型 RPG！自動戰鬥、收集裝備、轉生變強！
```

## 🚀 發布流程

### 步驟 1：建置（Build）

```bash
# 安裝 EAS CLI（如果還沒有）
npm install -g eas-cli

# 登入 Expo 帳號
eas login

# 配置專案
eas build:configure

# 建置 Android 生產版本
eas build --platform android --profile production
```

### 步驟 2：測試

- 下載 AAB 檔案
- 使用內部測試軌道（Internal Testing）測試
- 邀請 5-10 位測試者

### 步驟 3：提交

```bash
# 提交到 Google Play
eas submit --platform android
```

### 步驟 4：Google Play Console 設定

1. 填寫商店資訊
2. 上傳截圖和圖片
3. 設定內容分級
4. 設定定價（免費）
5. 選擇發布國家/地區
6. 提交審核

### 步驟 5：等待審核

- 通常 1-3 天
- 首次發布可能需要更長時間

## 🔄 發布後的更新流程

### OTA 更新（推薦，超快！）

```bash
# 發布更新（玩家下次啟動時自動下載）
eas update --branch production --message "修復 bug 和數值平衡"
```

**適用於：**
- 遊戲數值調整
- UI/UX 改進
- Bug 修復
- 新功能（不需要新權限）

### 完整更新（較慢）

```bash
# 建置新版本（記得更新版本號！）
eas build --platform android --profile production

# 提交更新
eas submit --platform android
```

**需要時：**
- 添加原生模組（廣告、IAP）
- 需要新權限
- 重大版本更新
