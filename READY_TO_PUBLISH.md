## 💡 重要提示

### OTA 更新的威力

發布後，90% 的更新可以透過 OTA 完成：

```bash
# 修改程式碼後
eas update --branch production --message "平衡調整和 bug 修復"

# 玩家下次啟動自動更新，無需審核！
```

**適用於：**
- ✅ 遊戲數值調整
- ✅ UI 改進
- ✅ Bug 修復
- ✅ 新功能（不需要新權限）
- ✅ 圖片/資源更新
- ✅ 文字修正

**不適用於：**
- ❌ 添加原生模組（廣告、IAP）
- ❌ 需要新權限
- ❌ 更改 App 名稱/圖示

### 為什麼快速發布是對的？

1. **真實用戶反饋 > 猜測**
   - 你現在不知道玩家會喜歡什麼
   - 數據會告訴你該優化什麼

2. **OTA 更新超快**
   - 字面意思就像 `git push`
   - 幾分鐘內推送給所有玩家

3. **避免過度打磨**
   - 完美主義會讓你永遠無法發布
   - 先發布 MVP，根據反饋改進

4. **時間就是機會**
   - 早發布 = 早獲得用戶
   - 早獲得用戶 = 早獲得反饋
   - 早獲得反饋 = 更快改進
